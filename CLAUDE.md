## Project Overview

AutoService PWA — a multi-tenant CRM for auto service shops (Ukrainian language). Masters (mechanics) manage clients, cars, orders, work items, and finances. An admin panel manages master registration via invite links.

## Tech Stack

- **Frontend:** Vanilla JS (no framework, no bundler, no npm). All JS is loaded as `<script>` tags in index.html/admin.html.
- **Backend:** Supabase (PostgreSQL + Auth + PostgREST). No server-side code.
- **Hosting:** GitHub Pages, deployed via GitHub Actions on push to `main`.
- **Auth:** Supabase Auth (email/password). JWT tokens stored in localStorage under `as_session`.

## Development

There is no build step, package manager, or test suite. To develop locally:

1. Fill in `config.js` with real `SUPABASE_URL` and `SUPABASE_ANON_KEY` values (do not commit).
2. Serve the root directory with any static server (e.g. `python3 -m http.server 8000`).
3. Open `http://localhost:8000` for the master app, `http://localhost:8000/admin.html` for admin.

Config values are injected in CI by `scripts/inject-config.py` using GitHub Actions secrets (`SUPABASE_URL`, `SUPABASE_ANON_KEY`, `ADMIN_PASSWORD`).

Database migrations live in `supabase/migrations/` and are applied manually in Supabase Dashboard SQL Editor.

## Architecture

### Two Entry Points

- **`index.html`** — Master-facing SPA. Screens: login, dashboard, clients, client-detail, orders, order-detail, finance, settings.
- **`admin.html`** — Admin panel. Manages masters list, generates invite links, blocks/unblocks masters.

### JS Module Pattern (Global Singletons)

Every JS file exports a global object singleton. There are no ES modules or imports. Load order in `index.html` matters:

```
config.js → js/db.js → js/onboarding.js → js/auth.js → js/clients.js → js/orders.js → js/finance.js → js/docs.js → js/app.js
```

Key singletons:
- **`DB`** (`js/db.js`) — Supabase REST client. Wraps fetch calls to PostgREST. All data operations go through `DB._get`, `DB._post`, `DB._patch`, `DB._delete`. Initialized with `DB.init(url, anonKey)`.
- **`Auth`** (`js/auth.js`) — Handles Supabase Auth login/register/refresh/logout. Stores JWT in localStorage. Sets `DB._authToken` for authenticated requests.
- **`App`** (`js/app.js`) — Screen navigation, settings, toast notifications. Entry point: `App.init()` on DOMContentLoaded.
- **`Clients`**, **`Orders`**, **`Finance`**, **`Docs`** — Domain modules. Each manages its own screen rendering and CRUD via `DB`.
- **`Onboarding`** (`js/onboarding.js`) — Handles `?invite=TOKEN` URL params for new master registration flow.

Admin panel loads: `config.js → js/db.js → admin/admin-db.js → admin/admin.js`. `AdminDB` is a thin wrapper around `DB` for admin actions.

### Data Model & Ownership Chain

```
masters ← clients ← cars ← orders ← order_items
masters ← finance
masters ← invites (admin-managed)
```

Each master sees only their own data. Ownership flows through foreign keys:
- `clients.master_id` → direct ownership
- `cars.client_id` → owned via client
- `orders.car_id` → owned via car → client
- `order_items.order_id` → owned via order → car → client
- `finance.master_id` → direct ownership

### Row Level Security (RLS)

RLS policies in `supabase/migrations/20260331_rls_fix.sql` enforce tenant isolation at the database level using `auth.uid()` matched against `masters.user_id`. Admin role is checked via `raw_user_meta_data->>'role' = 'admin'`.

### Screen Navigation

Screens are `<div class="screen">` elements toggled via `.active` class. Navigation is handled by `App.showScreen(id)` and `App.navigate(tab)`. Modals use `.modal-overlay` with `.open` class.

## Conventions

- All UI text is in **Ukrainian**. Keep all user-facing strings in Ukrainian.
- CSS uses **CSS custom properties** defined in `:root` in `css/app.css`. Dark theme only (bg: `#1a1a2e`, accent: `#e94560`).
- No third-party libraries — everything is vanilla JS and CSS.
- UUIDs are used for all entity IDs (generated by Supabase/PostgreSQL `gen_random_uuid()`).
- Order statuses: `new`, `in_progress`, `done`, `closed`.
- Finance types: `income`, `expense`.
- `Docs.generateAct()` generates a printable HTML document (Ukrainian legal format) opened in a new tab.
- Service Worker (`sw.js`) caches static assets with a cache-first strategy. API calls to Supabase are not cached.
